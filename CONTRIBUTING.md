# Contributing Guidelines

We appreciate and welcome contributions to the Personify project. By contributing, you can help us improve and enhance the functionality of our open-source project. Here are some guidelines to follow when contributing to Personify:

## Getting Started
1. Fork the Personify repository on GitHub.
2. Clone your forked repository to your local machine.
   ```
   git clone https://github.com/your-username/personify.git
   ```
3. Create a new branch for your feature or bug fix.
   ```
   git checkout -b feature/your-feature-name
   ```
4. Make your changes and commit them with descriptive commit messages.
   ```
   git commit -m "Add feature XYZ" 
   ```
5. Push your changes to your forked repository.
   ```
   git push origin feature/your-feature-name
   ```
6. Open a pull request in the Personify repository. Be sure to provide a clear and descriptive title and description for your pull request.

## Contribution Guidelines
- Follow the coding style and conventions used in the project.
- Ensure your code is well-documented.
- Write clear and concise commit messages.
- Test your changes thoroughly before submitting a pull request.
- Keep your pull requests focused and limited to a single feature or bug fix.
- Be respectful and considerate towards other contributors.

## Getting Help
If you need any assistance or have any questions regarding the contribution process or any aspect of the project, feel free to reach out to us by [creating an issue](https://github.com/ducconit/personify/issues) or contacting the project team at [ducconit@gmail.com](mailto:ducconit@gmail.com).
Thank you for your interest in contributing to Personify! We appreciate your support and look forward to your contributions.
